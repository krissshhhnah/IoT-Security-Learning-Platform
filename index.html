// ATTACK DIRECTORY

17 OSI-aligned
IoT attack vectors

Explore specific vulnerabilities and launch simulation nodes directly into the sandbox to study telemetry response and event logging.

01Data Link
Man-in-the-Middle

Intercepts transit frames between gateway and sensors to manipulate payloads or spoof active commands.

02Physical
Eavesdropping

Sniffs raw RF carrier signals over unencrypted wireless links to capture credential hashes and device configs.

03Network
Replay Attack

Captures valid authentication tokens and broadcasts them later to bypass hardware verification schemes.

04Data Link
Spoofing Attack

Forges MAC addresses of trusted IoT devices to gain illegal network entry and bypass access control lists.

05Network
Packet Injection

Crafts and injects malicious hex payloads into active routing queues to trigger remote shell overrides.

06Transport
Denial of Service

Floods the local IoT gateway router with junk request frames, causing CPU overhead and memory exfiltration.

07Transport
Distributed DoS

Inundates target router queues using multiple compromised zombie nodes, causing severe network outages.

08Physical
Jamming Attack

Emits high-intensity RF carrier noise to flood the radio spectrum, causing 100% wireless packet drop rate.

09Application
Credential Theft

Exfiltrates password databases and device cryptographic handshakes from insecure local storage nodes.

10Session
Session Hacking

Hijacks active BLE connection states or session keys, tricking smart devices into accepting illegal control packets.

11Physical
Rogue Insertion

Injects unverified rogue hardware modules into active physical connections, bypassing local firewalls.

12Network
Routing Attack

Manipulates mesh routing tables (RPL) to blackhole payloads or reroute data paths through threat nodes.

13Network
Sybil Attack

Spawns multiple pseudonymous virtual clone nodes to flood mesh routing algorithms and forge voting consensus.

14Presentation
Sensor Tampering

Injects forged telemetry data (light/temperature values) to trigger false system shutdowns.

15Physical
Timing Attack

Uses side-channel exfiltration to monitor microcontroller crypto execution cycles and extract private keys.

16Physical
Physical Access

Taps target board JTAG/UART headers to probe raw memory dumps or flash malicious firmware directly.

17Transport
Delay Attack

Holds back and buffers packet delivery times to compromise control-loop synchronization and timing parameters.

// IMPACT MATRIX

Security impact
in the real world

Learn how cascading IoT vulnerabilities disrupt mission-critical infrastructure environments.

HIGH THREAT
Physical Infrastructure Damage

Manipulated sensor telemetry and delayed commands can trigger mechanical failure in water treatment facilities, smart generators, and assembly line robots.

→ Sensor Tampering, Delay Attack, Rogue Insertion
MEDIUM THREAT
Confidential Data Exfiltration

Unencrypted wireless communication enables RF eavesdropping, allowing malicious actors to sniff authentication keys, usernames, and sensor tracking records.

→ Eavesdropping, MITM, Credential Theft
CRITICAL THREAT
Network Failure & Spectrum Denial

Continuous RF spectrum jamming or flood-based Denial of Service renders network nodes unreachable, completely shutting down smart monitoring queues.

→ Jamming Attack, DoS, DDoS
